# 「還我錢」互動網頁 - 計劃書

## 一、整體流程

1. **進入頁面**：頁面載入後彈出一個視窗（Modal），同時播放音效，視窗內容顯示斗大文字繁體中文「你欠我錢」，並有一個按鈕「查看詳情」。
2. **點擊「查看詳情」**：關閉彈出視窗，顯示「選擇身分」畫面，呈現三個按鈕：**媽 / 爸 / 妹**。
3. **點擊任一身分**：進入該人物的「欠款明細頁」，顯示：
   - 該人物欠款列表（可能有多筆，每筆一張卡片）
   - 每筆欠款包含：
     - 日期
     - 分類（例如：代墊消費 / 答應支付未付 / 借款 等）
     - 具體項目說明（買了什麼/做了什麼）
     - 金額
     - 付款證明（圖片，例如收據、轉帳截圖、聊天紀錄截圖）
   - 該人物欠款**總額**（自動加總）
4. **頁面有按鈕連去**：顯示收款資訊區塊：
   - 我的 QR Code（圖片）
   - 電話號碼
   - 銀行帳號

## 二、頁面結構（單一 HTML 檔案，方便直接開啟使用）

```
index.html
├── 內嵌 <style>：版面、配色、卡片、按鈕動畫
├── 內嵌 <script>：
│   ├── 欠款資料（JS 物件/陣列，方便您之後自行修改假資料 → 真資料）
│   ├── 畫面切換邏輯（彈窗 → 選人 → 明細）
│   └── 音效播放（用您提供的 mp3/wav）
├── Modal 彈窗區塊
├── 選擇身分區塊（三按鈕，可用各自的大頭照當按鈕背景）
├── 明細顯示區塊（依選擇動態渲染卡片）
└── 收款資訊區塊（QR Code + 電話 + 銀行帳號）
```

## 三、資料欄位設計（每筆欠款）

```js
{
  person: "媽",          // 媽 / 爸 / 妹
  date: "2026-01-15",    // 日期
  category: "代墊消費",   // 分類：代墊消費、答應支付未付、借款...
  item: "幫忙買的XX",     // 具體項目說明
  amount: 500,            // 金額（先用假資料）
  evidence: "evidence/xxx.jpg" // 付款證明圖片路徑（先放佔位圖）
}
```

- 「分類」我先預設幾種，您之後可以增減：
  - `代墊消費`：您先幫對方付錢
  - `養女基金`：對方答應付您的消費卻沒付
  - `借款`：直接借的錢

## 四、視覺風格

- 按鈕背景：可放「您的臉」或「對方的臉」當按鈕底圖（先用佔位圖，您之後替換成真實照片檔案）。
- 整體風格走「惡搞 / 趣味討債」感，例如紅色警示色 + 誇張字體 + 跳動動畫。
- 全站文字一律使用**繁體中文**。

## 五、待確認的最後幾項細節

1. **音效檔案**：請提供 mp3/wav 檔案（檔名或路徑），或先告知檔名，我建立資料夾結構，您再把檔案放進去。("D:\project\Own My Money!\SE")
2. **大頭照/QR Code 圖片**：同上，先用佔位圖框出位置，您之後把圖片檔案放入指定資料夾（例如 `images/mom.jpg`, `images/dad.jpg`, `images/sis.jpg`, `images/qrcode.jpg`, `images/me.jpg`）即可生效，不需改程式碼。("D:\project\Own My Money!\QRcode")
3. **假資料內容**：我先幫每人各放 2-3 筆「示意用」假欠款資料（含分類、項目、金額、日期），您之後直接在 HTML 裡的資料區塊修改成真實資料即可。
4. 確認電話號碼/銀行帳號也先用假資料佔位（例如 `0900-000-000` / `銀行：OO銀行 (000) 帳號：000-000-000000`），之後您自行替換。

## 七、第二輪更新需求（2026-06-20）

1. **配色**：改用彩虹亮色系（紅／橙／黃橙／黃／淺綠／綠／青／藍 八色漸層），整體風格更鮮豔活潑。
2. **收款方式**：Alipay / 轉數快 各加一個「點擊進入」按鈕，QR Code 圖片可點擊放大（燈箱 Lightbox）。
3. **欠款證明圖**：目前先全部套用同一張 QR Code 圖片當佔位證明圖，之後您再換成真實截圖。
4. **背景音樂（BGM）**：新增持續播放的背景音樂（與彈窗音效分開），需要您提供音樂檔案。
5. **身分驗證 + PIN 碼**：
   - 點擊「媽/爸/妹」後，先彈窗問「是否本人？」
   - 選「是」→ 若該身分**尚未設定 PIN**，讓他自行設定一個 PIN 碼（之後存於瀏覽器 `localStorage`）；若**已設定 PIN**，要求輸入正確 PIN 才能進入明細頁。
   - 製作者（您）需要有「萬能密碼／開發者模式」可以跳過 PIN 直接進入任何人的明細頁。
6. **環境設定（⚙ Settings 面板）**：
   - 各身分可自選「主題色」（並記住）。
   - 每筆欠款新增「我已付款」按鈕 → 點擊後彈出「請等待確認」，該筆資料變成「待確認」狀態（資料文字變半透明 + 加鎖圖示），直到您（製作者）以萬能密碼登入後手動「確認收款」，才恢復正常顯示並標記為已還款。
   - 設定面板內加「音量控制」滑桿（同時控制 BGM 與音效音量）。
7. **文字語言調整**：彈窗文字改用**粵語口語**：
   > 「你欠我錢呀！！依家係時候要面對現實啦，撳下面個掣睇詳情。」

## 八、需要您確認的細節

1. **PIN 碼安全性**：因為是純前端 HTML（無後端伺服器），PIN 只能存在瀏覽器 `localStorage`，技術上懂開發者工具的人仍可查看/繞過。這只能做「防一般人」程度的鎖，您是否接受？
2. **萬能密碼**：請提供一個您自己的萬能密碼（建議不要用太簡單的數字），用來：① 跳過任何人的 PIN 直接查看明細 ② 確認/解鎖「待確認」的還款狀態。
3. **「我已付款」按鈕範圍**：是每一筆欠款各自有獨立的「我已付款」按鈕（部分還款），還是整個人的總欠款只有一個「全部已付款」按鈕？
4. **BGM 音樂檔案**：請提供檔名或放入新建的 `audio/` 資料夾（彈窗音效 `SE/popup.mp3` 維持不變，BGM 另外處理）。
5. **PIN 是否要分人各自獨立**：媽、爸、妹三人 PIN 各自不同、互不相通，對嗎？
